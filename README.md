# zeroarm-ros2-can

 一个基于 **ROS2 + MoveIt2 + ros2_control + CAN 总线** 的机械臂真实控制项目。

<video src="assets/demo.mp4" controls width="600"></video>



---

## 项目简介

本项目面向自研 ZeroArm 机械臂，完成了从上层运动规划到底层真实硬件执行的完整控制链路，主要包括：

- 使用 MoveIt2进行运动规划
- 使用 ros2_control进行控制器与硬件接口管理
- 使用自定义 CAN 驱动层与 EMM V5 电机驱动器通信
- 支持真实关节位置 / 速度反馈读取
- 支持零点标定与参数化配置
- 支持对底层 `FD` 位置模式命令进行优化，减少重复发送带来的速度建立问题

当前版本采用的是：

> **上位机（Host-PC）控制架构**

而不是 MCU 本地控制架构。

## 系统结构

当前控制链路如下：

**MoveIt2**  
→ **ros2_control 控制器**  
→ **自定义硬件接口 `emm_v5_hardware`**  
→ **CAN 通信层 `emm_v5_driver`**  
→ **EMM V5 电机驱动器**  
→ **机械臂各关节**

当前主要使用 EMM V5 的 **位置模式 `FD`**，命令中包含：

- 目标位置
- 目标速度
- 加速度参数
- 真实位置 / 真实速度反馈

## 关键调参说明

项目中对运动效果影响较大的参数主要有以下几类。

### 1. 驱动器速度 / 加速度相关参数
例如：

- 电机目标 RPM / 速度上限
- `acc_level`
- 最小命令速度
- 死区相关参数

### 2. `acc_level`
这是 EMM V5 位置模式里非常关键的参数。

- `acc_level = 0`  
  更接近直接按目标速度运行
- `acc_level > 0`  
  驱动器内部会使用加减速曲线

实际调试中，不同关节往往需要不同的 `acc_level`。  
某些关节设为 0 可能会更硬、更响、更抖；设为较小非零值则会更平滑。

### 3. 供电电压
供电电压会影响高速运动能力。  
实际使用中，较低电压下的动态表现通常弱于更高电压供电条件。

### 4. 底层命令发送策略
如果同一条位置命令被频繁重复发送，会影响速度建立过程。  
本项目在硬件接口层对 `FD` 命令做了锁存与去重，以减少重复触发带来的问题。
